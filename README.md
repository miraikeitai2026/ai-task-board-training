# 🚀 グループToDo管理アプリ (Student Edition)

このアプリは、グループ活動での進捗状況やイベントまでの残り日数を「見える化」するための、演習用ToDoボードです。

## 🌟 この演習のポイント
- **データの構造を学ぶ**: `json` ファイルを編集することで、Webアプリの表示が変わる体験をします。
- **GitHubでの共同作業**: 自分の担当フォルダを編集し、チームで進捗を管理します。
- **AI活用の準備**: 改善案をメモし、将来的にAIを使って機能を拡張する基礎を作ります。

---

## 🛠 教員・管理者の方へ（演習前の準備）

演習を開始する前に、以下の手順で各グループ（01〜20）のフォルダを一括作成してください。

1.  このリポジトリをローカルPCにクローン（またはダウンロード）します。
2.  ルートディレクトリにある `create_groups.py` を実行します。
    ```bash
    python create_groups.py
    ```
3.  `groups/` フォルダ内に `group01` 〜 `group20` までのフォルダと初期ファイルが自動生成されます。
4.  生成されたファイルを GitHub に `add`, `commit`, `push` してください。
5.  リポジトリの **Settings > Pages** で、`main` ブランチを公開設定にします。

---

## 📖 学生のみなさんへ（演習の進め方）

自分のグループのToDoボードを表示・更新するために、以下の手順で作業を行ってください。

### 1. 自分のグループフォルダを探す
`groups/groupXX/`（XXは自分の班番号）があなたのチームの専用フォルダです。他のグループのフォルダは編集しないように注意しましょう。

### 2. イベント情報を設定する (`config.json`)
自分たちのグループ名や、目標とするイベントの日程を設定します。
- `groupName`: 班の名前
- `eventName`: 目標（例：「中間発表」「プロトタイプ完成」）
- `eventDate`: 目標の日付（`"2026-08-01"` のような形式）

### 3. タスクを登録・更新する (`tasks.json`)
自分たちがやるべきタスクを書き込みます。
- **status**: `未着手`, `作業中`, `完了` のいずれかを入力してください。これ以外の文字を入れると色が正しく表示されません。

### 4. 改善案を考える (`improvement_ideas.md`)
「ここが使いにくい」「こんな機能がほしい」と思ったら、このファイルにメモしてください。

---

## ⚠️ JSON編集の重要ルール
JSONファイルは1文字でも書き間違えると、アプリが動かなくなります。以下のルールを必ず守りましょう。
1. **すべて半角**で入力する（全角のスペースやカンマは厳禁！）。
2. 文字や項目は **ダブルクォーテーション `"`** で囲む。
3. 項目と項目の間は **カンマ `,`** で区切る。
4. **リストの最後の項目の後ろにはカンマを付けない。**

GitHub Pagesにアクセスし、自分のグループを選択して内容が反映されているか確認しましょう。

`https://<ユーザー名>.github.io/<リポジトリ名>/app/index.html`

## 📂 フォルダ構成
```text
.
├── app/                 # アプリ本体（教員が管理・学生は原則触りません）
│   ├── index.html       # 画面の構造
│   ├── style.css        # デザイン
│   └── script.js        # データを読み込むプログラム
├── groups/              # 【学生が編集する場所】
│   ├── group01/
│   │   ├── config.json  # グループ名や目標日の設定
│   │   ├── tasks.json   # タスクの一覧データ
│   │   └── improvement_ideas.md # 改善案のメモ
│   └── ...（group20まで）
├── create_groups.py     # 【教員用】フォルダ一括作成スクリプト
└── README.md            # この説明書
```
